# 回声迷宫：记忆挑战模式 (Memory Challenge Mode)

> 从「双人协作」转变为「单人记忆挑战」玩法重构

---

## 🎮 核心玩法

### 1. 单人游戏，周期性视野

| 状态 | 持续时间 | 视觉效果 | 游戏性 |
|------|----------|----------|--------|
| **光明期** | 3秒 | 完整显示迷宫、墙壁、出口、玩家位置 | 观察、记忆、规划路线 |
| **黑暗期** | 2秒 | 完全黑暗，**仅显示玩家位置**（发光蓝点） | 凭记忆移动，风险决策 |

### 2. 记忆挑战机制

**光明期必须记住：**
- 前方路径是否通畅（有无墙壁）
- 出口的大致方向和距离
- 周围几个格子的安全区域

**黑暗期必须决策：**
- 是否向前移动？
- 移动几步？（黑暗期可以连续移动）
- 风险：撞墙惩罚

### 3. 惩罚机制

| 行为 | 惩罚 |
|------|------|
| **撞墙** | 眩晕 2 秒（无法移动，黑暗/光明继续流逝）|
| **时间耗尽** | 60秒内未到达出口 = 游戏结束 |
| **步数耗尽** | 60步用完 = 游戏结束 |

### 4. 胜利条件

在 **60步内、60秒内**，到达迷宫右下角出口。

### 视野状态条

- **位置**：画布下方，明显但不遮挡
- **样式**：
  - 光明期：青色/绿色填充，发光效果
  - 黑暗期：深色背景，红色倒计时
- **显示**：剩余时间数字（精确到0.1s）

### 画布视觉效果

**光明期（3秒）：**
- 完整显示迷宫、墙壁、出口、玩家
- 墙壁：霓虹紫/青色发光线条
- 出口：金色脉冲发光
- 玩家：蓝色能量核心

**黑暗期（2秒）：**
- 画布变为纯黑（或极暗）
- **仅玩家可见**：发光的蓝色圆点（带光晕）
- 可选：上一帧看到的出口位置以 5% 透明度残留（帮助记忆）

## 🔧 技术实现

### 新增状态变量

```javascript
const VISIBILITY_CYCLE = {
  VISIBLE: 3000,  // 3秒光明
  DARK: 2000      // 2秒黑暗
};

state.visibility = {
  mode: 'VISIBLE',  // 'VISIBLE' | 'DARK'
  timer: 0,         // 当前模式已持续时间
  nextSwitchAt: 0   // 下次切换的时间戳
};
```

### 渲染逻辑修改

```javascript
function render() {
  // 更新视野状态
  updateVisibilityCycle();
  
  // 根据状态渲染
  if (state.visibility.mode === 'VISIBLE') {
    drawMazeFull();      // 完整迷宫
  } else {
    drawMazeDark();      // 仅玩家光点
  }
  
  // 更新HUD
  updateVisibilityBar();
}
```

### 输入控制

- **移动**：WASD / 方向键（不改变）
- **重置**：R 键（快速重新开始）
- **暂停**：空格键（可选）
