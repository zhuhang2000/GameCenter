## 📁 目录规范

```
discord-games/
├── auto-coding/              # 所有游戏项目
│   ├── snake-game/
│   ├── tetris-game/
│   └── plan/                 # 所有计划文档
│       ├── snake-plan.md
│       └── tetris-plan.md
├── memory/                   # 会话总结、每日笔记
│   ├── 2026-03-01-session.md
│   └── 2026-03-01.md
└── AGENTS.md                 # 本文件
```

**铁律：**
1. 游戏必须放在 `auto-coding/`，禁止放根目录
2. 计划文档必须放在 `auto-coding/plan/`
3. 每次 `/new` 后必须写会话总结到 `memory/`

---

## 🔄 /new 命令处理流程

当用户执行 `/new` 时：

1. **保存当前会话总结**
   - 文件名：`memory/YYYY-MM-DD-session-{number}.md`
   - 内容包括：本次完成的工作、关键决策、待办事项

2. **新开一轮会话**
   - 清理当前上下文
   - 准备接收新指令

3. **自动读取记忆**
   - 读取 `memory/` 下的今日总结
   - 保持上下文连续性

## 📝 工作流

1. **需求/点子** → 使用 `brainstorming` 扩展想法
2. **制定计划** → 写入 `auto-coding/plan/{project}-plan.md`
3. **开发实现** → 使用 `coding-agent` 按计划编码
4. **测试验证** → 本地测试、迭代优化
5. **提交归档** → `git commit`，更新文档
